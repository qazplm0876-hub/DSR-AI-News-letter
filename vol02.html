2025년 초, AI 연구자 안드레이 카르파티(Andrej Karpathy)가 처음 명명한 개념입니다. 코드를 몰라도, 말로 지시하면 AI가 실제로 작동하는 소프트웨어를 만들어줍니다.

Cursor, Windsurf, Claude Code 같은 도구에 자연어로 명령을 입력하면 AI가 코드를 작성하고, 파일을 만들고, 실행까지 합니다.

누구나 시스템을 만들 수 있다는 것은, 검증되지 않은 시스템이 조직 안에 퍼질 수 있다는 뜻이기도 합니다. DSR 전산팀은 바이브 코딩 활용 수칙을 깃허브에 수립하고, 현재 사내 바이브 코딩은 이 수칙 하에 진행되고 있습니다.
01
TEMPLATE · 개발 기준
DSR 깃허브의 템플릿 기반으로 개발할 것
전산팀이 사전에 검토한 DSR 깃허브 템플릿을 기반으로 개발을 시작합니다. 검증된 구조에서 출발해야 보안과 유지보수가 가능합니다.
02
DOCUMENTATION · 문서화
프로젝트 설명서에 목적·실행방법·담당자를 기재할 것
AI가 만든 도구라도 목적이 무엇인지, 어떻게 실행하는지, 담당자가 누구인지를 반드시 문서로 남겨야 합니다. 담당자가 바뀌어도 인수인계가 가능해야 합니다.
03
SECURITY · 보안
API Key 등 민감정보를 코드에 포함하지 않을 것
비밀번호·API Key 등 민감한 접속 정보를 코드 안에 직접 넣으면 외부에 노출될 수 있습니다. 반드시 별도 관리 방식을 사용해야 합니다.
Action Lab: 오늘의 지휘 미션
우리 팀에 필요한 도구,
바이브 코딩으로 만들 수 있을까

직접 만들지 않아도 됩니다. "우리 팀에 이런 게 있으면 좋겠다"는 과제를 정의하는 것이 리더의 역할입니다. 아래 미션을 통해 우리 팀의 바이브 코딩 과제를 발굴해보십시오.

1. 담당 부서 선택
2. Context (전략적 목표)
부서를 선택하면 전략적 맥락이 표시됩니다.
3. Secret Variable (리더의 현장 변수 주입)
⌨ Generated Agent Command
Level 5 Prompt
부서와 현장 변수를 입력하면 맞춤형 바이브 코딩 지휘 프롬프트가 자동 생성됩니다.
